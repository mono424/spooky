Deploying Sp00ky requires running the **Sp00ky Sidecar (SSP)** alongside your SurrealDB instance. For production deployments, the **Scheduler** coordinates multiple SSP instances.

## Architecture

1.  **SurrealDB**: Your persistent data store.
2.  **Sp00ky SSP (Sidecar)**: A backend service that monitors SurrealDB for changes and maintains materialized views.
3.  **Scheduler** *(optional)*: Central coordinator for distributed deployments with multiple SSPs. Maintains a RocksDB snapshot replica and WAL.
4.  **Client**: Connects directly to SurrealDB.

## Environment Variables

### SSP (Sidecar)

| Variable | Description | Default |
| :--- | :--- | :--- |
| `LISTEN_ADDR` | Address and port for the SSP HTTP server | `0.0.0.0:8667` |
| `SP00KY_AUTH_SECRET` | Bearer token for authenticating SSP API requests | *(required)* |
| `SURREALDB_ADDR` | SurrealDB host and port | `127.0.0.1:8000` |
| `SURREALDB_USER` | SurrealDB username | `root` |
| `SURREALDB_PASS` | SurrealDB password | `root` |
| `SURREALDB_NS` | SurrealDB namespace | `test` |
| `SURREALDB_DB` | SurrealDB database | `test` |
| `SCHEDULER_URL` | Scheduler URL (enables distributed mode) | *(unset)* |
| `SSP_ID` | Unique SSP identifier | `ssp-<uuid>` |
| `ADVERTISE_ADDR` | Externally reachable address for this SSP | *(unset)* |
| `HEARTBEAT_INTERVAL_MS` | Heartbeat interval in ms | `5000` |
| `SP00KY_CONFIG_PATH` | Path to `sp00ky.yml` for job config | `sp00ky.yml` |
| `TTL_CLEANUP_INTERVAL_SECS` | View TTL cleanup interval | `60` |

### Scheduler

| Variable | Description | Default |
| :--- | :--- | :--- |
| `SCHEDULER_ID` | Unique scheduler identifier | `scheduler-<uuid>` |
| `SP00KY_SCHEDULER_INGEST_HOST` | HTTP server bind host | `0.0.0.0` |
| `SP00KY_SCHEDULER_INGEST_PORT` | HTTP server bind port | `9667` |
| `SP00KY_SCHEDULER_DB_URL` | SurrealDB connection URL | `localhost:8000/rpc` |
| `SP00KY_SCHEDULER_DB_NAMESPACE` | SurrealDB namespace | `sp00ky` |
| `SP00KY_SCHEDULER_DB_DATABASE` | SurrealDB database | `sp00ky` |
| `SP00KY_SCHEDULER_DB_USERNAME` | SurrealDB username | `root` |
| `SP00KY_SCHEDULER_DB_PASSWORD` | SurrealDB password | `root` |
| `SP00KY_SCHEDULER_REPLICA_DB_PATH` | RocksDB snapshot replica path | `./data/replica` |
| `SP00KY_SCHEDULER_WAL_PATH` | Write-ahead log path | `./data/event_wal.log` |
| `SP00KY_SCHEDULER_LOAD_BALANCE` | Load balancing strategy | `least_queries` |
| `SP00KY_SCHEDULER_HEARTBEAT_INTERVAL_MS` | Expected heartbeat interval | `5000` |
| `SP00KY_SCHEDULER_HEARTBEAT_TIMEOUT_MS` | Heartbeat timeout before stale | `15000` |
| `SP00KY_SCHEDULER_BOOTSTRAP_TIMEOUT_SECS` | Bootstrap timeout | `120` |
| `SP00KY_SCHEDULER_SSP_POLL_INTERVAL_MS` | SSP health poll interval during bootstrap | `3000` |
| `SP00KY_SCHEDULER_MAX_BUFFER_PER_SSP` | Max buffered messages per SSP | `10000` |
| `SP00KY_SCHEDULER_SNAPSHOT_UPDATE_INTERVAL_SECS` | Snapshot update interval | `300` |
