**Spooky** is a reactive, local-first application framework built on top of **SurrealDB**. It bridges the gap between the database and the client, allowing for "Incantations" (LIVE queries) that update incrementally and efficiently.

## Supported Frameworks

Spooky is designed to be framework-agnostic, with bindings and examples for popular platforms:

- **TypeScript / JavaScript** (Vanilla, React, SolidJS)
- **Flutter** (Mobile/Desktop)
- **Rust** (WASM Modules)

## Key Features

<h3 class="flex items-center gap-3 text-xl font-semibold mt-8 mb-4">
  <Icon name="zap" class="w-6 h-6 text-zinc-400" />
  <span>1. Live Queries</span>
</h3>

Instead of fetching data once, your components subscribe to **Incantations**. When data changes on the server (or locally), your UI updates instantly. No manual refetching required.

<h3 class="flex items-center gap-3 text-xl font-semibold mt-8 mb-4">
  <Icon name="database" class="w-6 h-6 text-zinc-400" />
  <span>2. Local-First</span>
</h3>

Spooky uses IndexedDB to store a local replica of your data. This means your app works perfectly **offline**. Changes are queued and synchronized automatically when the connection returns.

<h3 class="flex items-center gap-3 text-xl font-semibold mt-8 mb-4">
  <Icon name="shield-check" class="w-6 h-6 text-zinc-400" />
  <span>3. End-to-End Type Safety</span>
</h3>

Your database schema _is_ your application schema. Spooky generates TypeScript interfaces directly from your `.surql` files, ensuring you catch errors at compile time, not runtime.
