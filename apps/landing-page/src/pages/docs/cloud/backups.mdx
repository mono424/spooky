Sp00ky Cloud provides built-in database backups for your SurrealDB data. Create manual snapshots, configure automatic schedules, and restore from any backup point.

## Quick Start

<CodeBlock
  code={`# Create a backup
spky cloud backup create

# List all backups
spky cloud backup list

# Restore from a backup
spky cloud backup restore <backup-id>`}
  lang="bash"
/>

## Creating Backups

Create a manual backup of your database:

<CodeBlock
  code={`# Create a backup
spky cloud backup create

# Create a named backup
spky cloud backup create --name before-migration`}
  lang="bash"
/>

Backups run asynchronously. The CLI returns immediately with a backup ID and status `pending`. The backup transitions to `in_progress` and then `completed` once done.

## Restoring from a Backup

<CodeBlock
  code={`spky cloud backup restore <backup-id>`}
  lang="bash"
/>

<Warning>
Restoring a backup replaces your current database contents with the backup data. Any changes made after the backup was created will be lost. Consider creating a fresh backup before restoring.
</Warning>

## Deleting Backups

<CodeBlock
  code={`spky cloud backup delete <backup-id>`}
  lang="bash"
/>

## Automatic Backups

Configure scheduled backups with a cron expression:

<CodeBlock
  code={`# Enable daily backups at 2 AM, keep last 7
spky cloud backup configure --enabled true --schedule "0 2 * * *" --retention 7

# Disable automatic backups
spky cloud backup configure --enabled false

# Change retention to keep 30 backups
spky cloud backup configure --retention 30`}
  lang="bash"
/>

| Option | Description |
| :--- | :--- |
| `--enabled` | Enable or disable automatic backups (`true`/`false`) |
| `--schedule` | Cron expression for backup frequency (e.g. `"0 2 * * *"` for daily at 2 AM) |
| `--retention` | Number of backups to keep before oldest are deleted |

## Database Reset

Reset your database to a clean state — drops all data and re-runs migrations:

<CodeBlock
  code={`spky cloud backup reset`}
  lang="bash"
/>

The CLI will:

1. **Warn** you that all data will be permanently deleted
2. **Ask for confirmation**
3. **Offer to create a backup** before resetting (recommended)
4. Wait for the backup to complete if created
5. **Reset the database** and re-apply migrations

To skip the pre-reset backup prompt:

<CodeBlock
  code={`spky cloud backup reset --no-backup`}
  lang="bash"
/>

<Warning>
Database reset is irreversible. All user accounts, records, and application data will be permanently deleted. Always create a backup first unless you're resetting a development project.
</Warning>
