Sp00ky Cloud projects live inside a **tenant**. Every tenant has an owner and can have additional team members who share access to projects, deployments, and the shared vault.

## Plan Limits

| Plan | Included Members | Additional Members |
| :--- | :--- | :--- |
| **Starter** | 1 (owner only) | +$5/mo each |
| **Pro** | Up to 5 | +$5/mo each |

Additional team members beyond the plan limit are billed as a metered add-on at $5 per member per month.

## Inviting Team Members

Invite a new member by their GitHub email address:

<CodeBlock
  code={`spky cloud team invite <email>`}
  lang="bash"
/>

The invited user receives an email with a link to accept the invitation. Once accepted, they can log in with `spky cloud login` and will have access to all projects within the tenant.

<Note title="GitHub Account Required">
Team members must have a GitHub account with the invited email address. Sp00ky Cloud uses GitHub OAuth for authentication, so the invited user will log in through the same <code>spky cloud login</code> flow.
</Note>

### Pending Invitations

View all pending invitations that have not yet been accepted:

<CodeBlock
  code={`spky cloud team invitations`}
  lang="bash"
/>

Revoke a pending invitation:

<CodeBlock
  code={`spky cloud team revoke <email>`}
  lang="bash"
/>

## Removing Team Members

Remove a member from the tenant:

<CodeBlock
  code={`spky cloud team remove <email>`}
  lang="bash"
/>

<Warning>
Removing a team member immediately revokes their access to all projects and the shared vault. Their vault keys are deleted and any API keys they created are revoked. This action cannot be undone — you will need to re-invite them if you want to restore access.
</Warning>

## Roles

| Role | Capabilities |
| :--- | :--- |
| **Owner** | Full access. Manage billing, invite/remove members, create/destroy projects, deploy, manage vault. |
| **Member** | Access all projects. Deploy, view logs, manage environment variables via shared vault. Cannot manage billing or remove other members. |

Each tenant has exactly one owner (the user who created the first project). Ownership cannot be transferred at this time.
