Sp00ky Cloud handles infrastructure provisioning, scaling, and management so you can deploy your project with a single command. No Docker Compose files, no server setup — just `spky cloud deploy`.

## Prerequisites

- The Sp00ky CLI installed (`npm i -g @sp00ky/cli`)
- A `sp00ky.yml` config file in your project root
- Docker installed locally (for building backend images)

## Quick Start

<CodeBlock
  code={`# 1. Log in to Sp00ky Cloud
spky cloud login

# 2. Create a cloud project
spky cloud create --slug my-app

# 3. Deploy
spky cloud deploy`}
  lang="bash"
/>

## Authentication

Log in via GitHub OAuth. The CLI opens your browser to complete the flow:

<CodeBlock
  code={`spky cloud login`}
  lang="bash"
/>

A 4-digit verification code is shown in your terminal. Confirm it in the browser to authenticate. Credentials are saved to `~/.sp00ky/credentials.json`.

To log out:

<CodeBlock
  code={`spky cloud logout`}
  lang="bash"
/>

<Note title="CI/CD">
For automated pipelines, create an API key with <code>spky cloud keys create</code> and set the <code>SPOOKY_API_KEY</code> environment variable. No browser login required.
</Note>

## Creating a Project

<CodeBlock
  code={`# Interactive — prompts for slug if not provided
spky cloud create

# With options
spky cloud create --slug my-app --plan starter`}
  lang="bash"
/>

Available plans: `starter`, `pro`, `business`. The slug must be lowercase alphanumeric with hyphens, and is globally unique.

Your project slug should match the `slug` field in your `sp00ky.yml`:

<CodeBlock
  code={`# sp00ky.yml
slug: my-app
# ... rest of config`}
  lang="yaml"
/>

## Deploying

<CodeBlock
  code={`spky cloud deploy`}
  lang="bash"
/>

The deploy command performs these steps automatically:

1. **Validates** your `sp00ky.yml` configuration
2. **Builds** Docker images for each backend (linux/amd64)
3. **Uploads** images to Sp00ky Cloud (skips unchanged images)
4. **Provisions** infrastructure (SurrealDB, Scheduler, SSP instances)
5. **Runs migrations** against the cloud database
6. **Deploys** your backend and frontend containers
7. **Configures** DNS and SSL certificates

You'll see real-time progress as each step completes.

### Upgrading Infrastructure

To also upgrade SSP and Scheduler to the latest version:

<CodeBlock
  code={`spky cloud deploy --upgrade`}
  lang="bash"
/>

## Checking Status

<CodeBlock
  code={`spky cloud status`}
  lang="bash"
/>

Shows the current deployment version, status, and all running VMs with their roles and versions.

## Status Badge

Add a live deployment status badge to your GitHub README or any webpage. The badge is a public SVG endpoint — no authentication required.

<CodeBlock
  code={`![Sp00ky Status](https://api.sp00ky.cloud/v1/badge/your-project-slug)`}
  lang="markdown"
/>

Or in HTML:

<CodeBlock
  code={`<img src="https://api.sp00ky.cloud/v1/badge/your-project-slug" alt="Sp00ky Status" />`}
  lang="html"
/>

The badge displays the current deployment status:

| Status | Color | Meaning |
| :--- | :--- | :--- |
| **live** | Green | Deployment is running |
| **building** | Yellow | Deployment in progress (pending, provisioning, migrating, or deploying) |
| **error** | Red | Deployment failed |
| **unknown** | Gray | No deployment found |

The badge is never cached — it always reflects the current state. You can also append `.svg` to the URL: `/v1/badge/your-project-slug.svg`.

<Note title="Staging">
For staging deployments, use the staging API URL: <code>https://api-stg.sp00ky.cloud/v1/badge/your-project-slug</code>
</Note>

## Viewing Logs

Tail logs from your cloud deployment in real time:

<CodeBlock
  code={`# All services
spky cloud logs

# Filter by service
spky cloud logs --filter surrealdb
spky cloud logs --filter ssp
spky cloud logs --filter backend
spky cloud logs --filter frontend

# Multiple services (comma-separated)
spky cloud logs --filter ssp,surrealdb

# Blueprint shorthand: "spooky" = ssp + scheduler
spky cloud logs --filter spooky

# Split view
spky cloud logs --filter ssp,surrealdb --split h   # horizontal (stacked)
spky cloud logs --filter ssp,surrealdb --split v   # vertical (side-by-side)`}
  lang="bash"
/>

## Scaling

Scale your SSP instances horizontally:

<CodeBlock
  code={`# Scale to 3 SSP instances
spky cloud scale --ssp 3`}
  lang="bash"
/>

## Listing Projects

<CodeBlock
  code={`spky cloud list`}
  lang="bash"
/>

Shows all your cloud projects with their plan, status, and creation date.

## Destroying a Project

<Warning>
This permanently tears down all VMs and infrastructure for the project. Data will be lost unless you have a backup.
</Warning>

<CodeBlock
  code={`spky cloud destroy`}
  lang="bash"
/>

## API Keys

Create long-lived API keys for CI/CD pipelines:

<CodeBlock
  code={`# Create a new key
spky cloud keys create

# List keys
spky cloud keys list

# Revoke a key
spky cloud keys revoke <key-id>`}
  lang="bash"
/>

Use the key in CI by setting the environment variable:

<CodeBlock
  code={`export SPOOKY_API_KEY=spk_live_...
spky cloud deploy`}
  lang="bash"
/>

## GitHub Integration

Link a GitHub repository for automated deployments on push:

<CodeBlock
  code={`# Set up auto-deploy
spky cloud link setup

# Check link status
spky cloud link status

# Configure settings
spky cloud link settings --branch main --auto-deploy true

# Manually trigger a deploy
spky cloud link trigger

# View recent build runs
spky cloud link runs

# Remove the link
spky cloud link unlink`}
  lang="bash"
/>
