Cloud Deployment

Sp00ky Cloud handles infrastructure provisioning, scaling, and management so you can deploy your project with a single command. No Docker Compose files, no server setup — just spooky cloud deploy.

Prerequisites

  • The Sp00ky CLI installed (npm i -g @sp00ky/cli)
  • A sp00ky.yml config file in your project root
  • Docker installed locally (for building backend images)

Quick Start

Bash 
# 1. Log in to Sp00ky Cloud
spooky cloud login

# 2. Create a cloud project
spooky cloud create --slug my-app

# 3. Deploy
spooky cloud deploy

Authentication

Log in via GitHub OAuth. The CLI opens your browser to complete the flow:

Bash 
spooky cloud login

A 4-digit verification code is shown in your terminal. Confirm it in the browser to authenticate. Credentials are saved to ~/.sp00ky/credentials.json.

To log out:

Bash 
spooky cloud logout
CI/CD

For automated pipelines, create an API key with spooky cloud keys create and set the SPOOKY_API_KEY environment variable. No browser login required.

Creating a Project

Bash 
# Interactive — prompts for slug if not provided
spooky cloud create

# With options
spooky cloud create --slug my-app --plan starter

Available plans: starter, pro, business. The slug must be lowercase alphanumeric with hyphens, and is globally unique.

Your project slug should match the slug field in your sp00ky.yml:

yaml 
# sp00ky.yml
slug: my-app
# ... rest of config

Deploying

Bash 
spooky cloud deploy

The deploy command performs these steps automatically:

  1. Validates your sp00ky.yml configuration
  2. Builds Docker images for each backend (linux/amd64)
  3. Uploads images to Sp00ky Cloud (skips unchanged images)
  4. Provisions infrastructure (SurrealDB, Scheduler, SSP instances)
  5. Runs migrations against the cloud database
  6. Deploys your backend and frontend containers
  7. Configures DNS and SSL certificates

You’ll see real-time progress as each step completes.

Upgrading Infrastructure

To also upgrade SSP and Scheduler to the latest version:

Bash 
spooky cloud deploy --upgrade

Checking Status

Bash 
spooky cloud status

Shows the current deployment version, status, and all running VMs with their roles and versions.

Status Badge

Add a live deployment status badge to your GitHub README or any webpage. The badge is a public SVG endpoint — no authentication required.

markdown 
![Sp00ky Status](https://api.sp00ky.cloud/v1/badge/your-project-slug)

Or in HTML:

HTML 
<img src="https://api.sp00ky.cloud/v1/badge/your-project-slug" alt="Sp00ky Status" />

The badge displays the current deployment status:

StatusColorMeaning
liveGreenDeployment is running
buildingYellowDeployment in progress (pending, provisioning, migrating, or deploying)
errorRedDeployment failed
unknownGrayNo deployment found

The badge is never cached — it always reflects the current state. You can also append .svg to the URL: /v1/badge/your-project-slug.svg.

Staging

For staging deployments, use the staging API URL: https://api-stg.sp00ky.cloud/v1/badge/your-project-slug

Viewing Logs

Tail logs from your cloud deployment in real time:

Bash 
# All services
spooky cloud logs

# Filter by service
spooky cloud logs --filter surrealdb
spooky cloud logs --filter ssp
spooky cloud logs --filter backend
spooky cloud logs --filter frontend

# Multiple services (comma-separated)
spooky cloud logs --filter ssp,surrealdb

# Blueprint shorthand: "spooky" = ssp + scheduler
spooky cloud logs --filter spooky

# Split view
spooky cloud logs --filter ssp,surrealdb --split h   # horizontal (stacked)
spooky cloud logs --filter ssp,surrealdb --split v   # vertical (side-by-side)

Scaling

Scale your SSP instances horizontally:

Bash 
# Scale to 3 SSP instances
spooky cloud scale --ssp 3

Listing Projects

Bash 
spooky cloud list

Shows all your cloud projects with their plan, status, and creation date.

Destroying a Project

Warning

This permanently tears down all VMs and infrastructure for the project. Data will be lost unless you have a backup.

Bash 
spooky cloud destroy

API Keys

Create long-lived API keys for CI/CD pipelines:

Bash 
# Create a new key
spooky cloud keys create

# List keys
spooky cloud keys list

# Revoke a key
spooky cloud keys revoke <key-id>

Use the key in CI by setting the environment variable:

Bash 
export SPOOKY_API_KEY=spk_live_...
spooky cloud deploy

GitHub Integration

Link a GitHub repository for automated deployments on push:

Bash 
# Set up auto-deploy
spooky cloud link setup

# Check link status
spooky cloud link status

# Configure settings
spooky cloud link settings --branch main --auto-deploy true

# Manually trigger a deploy
spooky cloud link trigger

# View recent build runs
spooky cloud link runs

# Remove the link
spooky cloud link unlink

Billing

Bash 
# Open the billing portal
spooky cloud billing

# View current month's usage
spooky cloud billing usage

CLI Reference

Bash 
spooky cloud <COMMAND>

Commands:
login              Authenticate with Sp00ky Cloud (opens browser)
logout             Clear stored credentials
create             Create a new cloud project
list               List cloud projects
deploy [--upgrade] Deploy current project to the cloud
status             Show deployment status
logs [--filter]    Tail logs from cloud deployment
scale --ssp N      Scale SSP instances
destroy            Destroy cloud project and all VMs
backup             Manage database backups
billing            Billing management
keys               Manage API keys for CI/CD
link               Link a GitHub repository for auto-deploy
env                Manage encrypted environment variables
